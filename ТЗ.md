# Техническое задание (ТЗ): Проект «Cangaroo Web» (v2.0 — Local-First)

## 1. Цель проекта

Создание интерактивной веб-платформы для обучения алгоритмизации. Ученик управляет исполнителем «Кенгуру» на координатной сетке, записывая команды текстовым кодом, что подготавливает его к переходу на C++.

## 2. Основные функциональные модули

- **Игровое поле (Phaser.js):**
  - Сетка 64x64 пикселя с киберпанк-дизайном.
  - Анимированный робот-кенгуру с 4 ракурсами движения.
  - Визуализация закрашенных клеток и препятствий.
- **Умный редактор кода (CodeMirror 6):**
  - Подсветка синтаксиса для кастомного DSL.
  - Автодополнение команд (IntelliSense).
  - Индикация синтаксических ошибок в реальном времени.
- **Интерпретатор и Очередь команд:**
  - Парсинг текстового кода в абстрактное синтаксическое дерево (AST).
  - Пошаговое выполнение действий в Phaser.js для плавной анимации.
- **Система прогресса (LocalStorage):**
  - Автоматическое сохранение пройденных уровней в браузере пользователя.
  - Отсутствие необходимости регистрации для быстрого старта на уроке.
- **Панель управления:**
  - Кнопки «Пуск», «Шаг», «Стоп», «Сброс».
  - Ползунок регулировки скорости анимации кенгуру.

## 3. Спецификация синтаксиса (DSL)

Используется румыноязычный синтаксис, близкий к C-style, для формирования навыков написания реального кода:

- **Команды:** `pas;`, `dreapta;`, `stanga;`, `vopseste;`, `sari;`.
- **Циклы:** `repetă (n) { ... }`.
- **Условия:** `dacă (liber) { ... }`.

## Полный стек технологий

### Фронтенд и UI

- **SvelteKit:** Основной фреймворк, оптимизированный под Vercel.
- **Tailwind CSS:** Стилизация в стиле Cyberpunk (Neon Cyan/Magenta).
- **shadcn-svelte:** Компоненты интерфейса (карточки уровней, кнопки).
- **Lucide Icons:** Технологичные иконки для интерфейса.

### Графика и Логика

- **Phaser.js:** Рендеринг игрового мира и анимация спрайтов.
- **CodeMirror 6:** Ядро текстового редактора.
- **Peggy.js:** Генератор парсера для обработки вложенных циклов `repetă`.

### Хранение и Деплой

- **LocalStorage API:** Сохранение состояния игры и прогресса уровней.
- **Vercel:** Хостинг для статического развертывания проекта.

## План разработки (MVP)

1.  **Настройка SvelteKit + Tailwind:** Базовая разметка страницы «Редактор + Поле».
2.  **Интеграция Phaser:** Отрисовка сетки и загрузка спрайта робота-кенгуру.
3.  **Создание Парсера:** Реализация базовых команд (`pas`, `dreapta`) через Peggy.js.
4.  **Связь Editor -> Game:** Выполнение написанного кода на игровом поле.
5.  **LocalStorage:** Реализация сохранения открытых уровней.
